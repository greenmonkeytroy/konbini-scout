## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can demo what you've built.
2. **Infrastructure before features.** The first phase sets up everything so feature work in later phases is fast and clean.
3. **Magic moment first.** The shared shelf experience works as early as Phase 2. Everything else builds on top.
4. **Test as you go.** Verify each task before marking it complete. Run the dev server after every phase.
5. **Progressive enhancement.** Start with the simplest working version, then layer on polish.
6. **Review before proceeding.** Push each completed phase as a PR and let a review agent (e.g. [CodeRabbit](https://coderabbit.ai)) review it before starting the next phase. External review catches issues the coding agent won't flag.
7. **Content depth enables launch.** This is a content-driven product. The database must feel rich and the illustrations must be polished before soft launch — don't rush to go live with a thin catalogue.

## Agent Session Guide

### How to Use This Roadmap with Your Coding Agent

1. **Start a session:** Give your coding agent the phase prompt at the top of each phase.
2. **Read selectively:** Each phase lists its Reference sections — the specific parts of the PRD and vision doc needed for that phase. Read only those, not the entire documents.
3. **Let it work:** The agent finds the first unchecked task, implements it, marks it complete, and moves to the next.
4. **One session = one phase (ideally):** Try to complete a full phase in one session for best continuity. If you stop mid-phase, the agent resumes from the first unchecked task.
5. **Push a PR for review:** When a phase is complete, push as a PR and let [CodeRabbit](https://coderabbit.ai) review it before starting the next phase. It's free for public repos.
6. **Phase 6 is different:** Check `[VALIDATE FIRST]` tasks with real usage data before building them. Don't build features no one asked for.

### Session Tips

- **Don't read everything:** The PRD and vision doc are comprehensive. Each phase's Reference sections tell the agent exactly what to read.
- **Don't skip tasks:** Tasks are ordered intentionally. Skipping creates dependency issues.
- **Verify after each phase:** Run the app and walk through the user flow manually before pushing.
- **Keep the status line updated:** After completing tasks, update `**Status:** X/62 tasks complete` and `**Current Phase:**` at the top of this file.
